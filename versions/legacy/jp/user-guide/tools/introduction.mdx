### ツールの定義

ツールはLLM（大規模言語モデル）の機能を拡張し、ネットワーク検索や科学計算、画像生成などの能力を持たせることで、LLMが外部世界と接続する能力を強化します。Difyは2種類のツールを提供しています：**組み込みツール**と**カスタムツール**。

Difyエコシステムが提供する組み込みツールを直接使用することも、簡単にカスタムAPIツール（現在はOpenAPI / SwaggerおよびOpenAI Plugin仕様をサポート）をインポートすることもできます。

#### ツールの役割：

1. ツールを使用することで、LLM上でより強力なAIアプリケーションを作成できます。たとえば、インテリジェントなアシスタント型アプリケーション（エージェント）に適切なツールをプログラムし、タスクの推論やステップの分解、ツールの呼び出しを通じて複雑なタスクを完了することが可能です。
2. 他のシステムやサービスとアプリケーションを簡単に接続し、外部環境とやり取りできます。コードの実行や専用情報源へのアクセスなどが可能です。

#### 組み込みツールの認証

Difyエコシステムが提供する組み込みツールを直接使用する場合、使用前に対応する認証情報を設定する必要があります。

<Frame caption="組み込みツールの認証設定">
  <img src="/ja-jp/img/configure-first-party-tool-api.png" alt="組み込みツールの認証設定" />
</Frame>

認証情報の検証が成功すると、ツールは「認可済み」の状態になります。認証情報を設定することで、ワークスペースのすべてのメンバーがこのツールを使用してアプリケーションをプログラムできるようになります。

<Frame caption="カスタムツールの作成">
  <img src="/ja-jp/img/en-tools-create-customized-tools-1.png" alt="カスタムツールの作成" />
</Frame>

### カスタムツールの作成方法

"ツール-カスタムツール"を使用して、カスタムAPIツールを簡単にインポートできます。現在、OpenAPI / SwaggerとChatGPTプラグインの仕様がサポートされています。OpenAPIスキーマの内容を直接コピーすることも、URLからインポートすることも可能です。OpenAPI / Swaggerの詳細については、[公式ドキュメント](https://swagger.io/specification/)をご覧ください。

ツールでは、2種類の認証方法をサポートしています：認証なしとAPIキーの2つです。

<Frame caption="カスタムツールのパラメータテスト">
  <img src="/ja-jp/img/en-tools-create-customized-tools-2.png" alt="カスタムツールのパラメータテスト" />
</Frame>

カスタム ツールを作成すると、ワークスペース内のすべてのメンバーが「スタジオ」でアプリケーションをオーケストレーションするときにそのツールを使用できるようになります。

<Frame caption="カスタムツールのパラメーターテスト">
  <img src="/ja-jp/user-guide/.gitbook/assets/image (199).png" alt="カスタムツールのパラメーターテスト" />
</Frame>

カスタムツールの作成が完了すると、ワークスペース内のすべてのメンバーが「スタジオ」でアプリケーションを構築する際にこのツールを使用できるようになります。

#### Cloudflare Workers

[dify-tools-worker](https://github.com/crazywoola/dify-tools-worker)を使うことで、カスタムツールを迅速にデプロイすることも可能です。このツールには以下の機能が含まれています：

* Difyのルート `https://difytoolsworker.yourname.workers.dev/doc` をインポートし、OpenAPI互換のインターフェースドキュメントを提供します。
* APIの実装コードを生成し、Cloudflare Workersに直接デプロイできます。
