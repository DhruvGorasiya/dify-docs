title: 插件调试
---

## 显示 Helm Chart 值

```bash
helm show values dify/dify
```

这是 Helm chart 值中插件调试配置的示例。

```yaml
plugin_daemon:
  serviceType: "NodePort"
  remoteInstallHost: "127.0.0.1"
  remoteInstallPort: "5003"
```

- `serviceType`
  - 用于公开 `plugin_daemon` 的服务类型。可以是 `NodePort` 或 `LoadBalancer`。
  - `NodePort`: 在每个节点的 IP 上以静态端口公开服务。
  - `LoadBalancer`: 在支持的云提供商中创建外部负载均衡器，将流量路由到 `plugin_daemon`。
  - 有关更多详细信息，请参阅 [Kubernetes Service Types 文档](https://kubernetes.io/docs/concepts/services-networking/service/#publishing-services-service-types)。

- `remoteInstallHost`: 这是 `Node` IP 或 `LoadBalancer` IP。
  - 如果使用 `NodePort`，则应为 Kubernetes 节点之一的 IP。
  - 如果使用 `LoadBalancer`，则应为负载均衡器的 IP。

- `remoteInstallPort`: 这是 `plugin_daemon` 监听调试的端口。
  - 如果使用 `NodePort`，则应为分配给服务的端口。
  - 如果使用 `LoadBalancer`，则应为负载均衡器公开的端口。
